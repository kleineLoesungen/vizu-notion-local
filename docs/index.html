vizu-notion-local

Visualize Your Notion Workspace

Render hierarchical or sequential Notion databases as interactive metro maps and process flows — no code changes required.

View on GitHub Docker Hub

Features

⚙

Config-Driven

Define Notion sources and column mappings in a JSON config file. Update the config, restart the container, and the app validates and reloads — no code changes.

◉

Two Visualization Types

Hierarchical databases render as metro-style maps. Sequential/workflow databases render as process flows. The app auto-detects which type is available for each source.

▶

Fully Local & Self-Hosted

Single docker-compose up command. Your Notion token stays server-side. No cloud relay, no external dependencies — run on your own machine.

See It In Action

Render your Notion data as interactive metro maps or process flows.

Metro Map — Milestones and phases rendered as transit-style timeline tracks

Flow Diagram — Notion databases rendered as navigable node-edge graphs

How It Works

01
Configure

Edit a simple JSON config file with your Notion database IDs and column mappings.

02
Run

Start the container with docker-compose up. The app validates your config and connects to Notion.

03
Visualize

Open the browser and view your data as interactive metro maps or process flows. Filter, export, and share.

Built with Nuxt + Vue 3 + TailwindCSS. Config-driven visualization for Notion. Metro map rendering powered by Metroviz.
