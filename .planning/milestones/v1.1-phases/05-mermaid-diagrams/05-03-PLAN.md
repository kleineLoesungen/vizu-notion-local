    **Changes required:**

    **1. Script section — add new reactive state and composable**

    Extend the existing destructuring from useSourceData() to include:
    ```typescript
    const { ..., hasMermaidTemplates, mermaidTemplates } = useSourceData(sourceId)
    ```

    Change activeVizType type to include 'mermaid':
    ```typescript
    const activeVizType = ref<'metro' | 'flow' | 'mermaid'>('metro')
    ```

    Add active template tracking:
    ```typescript
    const activeMermaidTemplateId = ref<string>('')

    function selectMermaidTemplate(templateId: string) {
      activeMermaidTemplateId.value = templateId
      activeVizType.value = 'mermaid'
    }
    ```

    Add useMermaidTemplate composable (only renders when activeVizType === 'mermaid'):
    ```typescript
    const mermaidDiagram = useMermaidTemplate(activeMermaidTemplateId)
    ```

    Update the existing watch that auto-selects viz type on load. The watch likely looks like:
    ```typescript
    watch([isMetroEligible, isFlowEligible], ([metro, flow]) => {
      if (!metro && flow) activeVizType.value = 'flow'
    }, { immediate: true })
    ```
    Do not break this — Mermaid is always user-initiated (click on template button), so no auto-selection needed for Mermaid.

    Also update the "not eligible" check to include Mermaid:
    ```typescript
    // The v-else-if for "Cannot visualize this source" — add hasMermaidTemplates to condition:
    // Change: v-else-if="!isMetroEligible && !isFlowEligible"
    // To: v-else-if="!isMetroEligible && !isFlowEligible && !hasMermaidTemplates"
    ```

    **2. Template section — viz type selector (lines ~150-175)**

    Replace the existing selector block. The new logic: if ANY of metro, flow, or mermaid templates are available AND there is more than one option → show interactive buttons; if only one option → show label. Simplest approach: always show buttons when count > 1, show single label when count === 1.

    In practice, replace:
    ```html
    <template v-if="isMetroEligible && isFlowEligible">
      <button ... @click="activeVizType = 'metro'">Metro Map</button>
      <button ... @click="activeVizType = 'flow'">Process Flow</button>
    </template>
    <span v-else ...>{{ activeVizType === 'metro' ? 'Metro Map' : 'Process Flow' }}</span>
    ```

    With:
    ```html
    <!-- Viz type toggle buttons — shown when multiple types are available -->
    <template v-if="isMetroEligible || isFlowEligible || hasMermaidTemplates">
      <button
        v-if="isMetroEligible"
        :class="['px-4 py-2 rounded text-sm font-medium', activeVizType === 'metro' ? 'bg-blue-600 text-white' : 'bg-gray-100 text-gray-700']"
        @click="activeVizType = 'metro'"
      >Metro Map</button>
      <button
        v-if="isFlowEligible"
        :class="['px-4 py-2 rounded text-sm font-medium', activeVizType === 'flow' ? 'bg-blue-600 text-white' : 'bg-gray-100 text-gray-700']"
        @click="activeVizType = 'flow'"
      >Process Flow</button>
      <!-- One button per Mermaid template that references this source (D-08) -->
      <button
        v-for="tmpl in mermaidTemplates"
        :key="tmpl.id"
        :class="['px-4 py-2 rounded text-sm font-medium', activeVizType === 'mermaid' && activeMermaidTemplateId === tmpl.id ? 'bg-blue-600 text-white' : 'bg-gray-100 text-gray-700']"
        @click="selectMermaidTemplate(tmpl.id)"
      >{{ tmpl.title }}</button>
    </template>
    ```

    Note: The old `v-else` single-label span can be removed — the new buttons handle the single-type case too (one button is fine).

    **3. Template section — viz render area**

    After the existing FlowDiagram template block (the v-else-if for flow), add a Mermaid case. The viz area currently has:
    ```html
    <MetrovizMap v-if="..." ... />
    <template v-else-if="isFlowEligible"> ... <FlowDiagram ... /> </template>
    ```

    Add after the flow template block:
    ```html
    <!-- Mermaid diagram (MERM-03) -->
    <template v-else-if="activeVizType === 'mermaid' && activeMermaidTemplateId">
      <LoadingSpinner v-if="mermaidDiagram.isLoading.value" />
      <ErrorAlert
        v-else-if="mermaidDiagram.fetchError.value"
        heading="Failed to load diagram data"
        :message="`Could not fetch data for diagram: ${mermaidDiagram.fetchError.value?.message ?? 'Unknown error'}. Check that sources are configured correctly in sources.json and Notion integration is accessible.`"
      />
      <ErrorAlert
        v-else-if="mermaidDiagram.renderError.value"
        heading="Failed to render diagram"
        :message="`Template rendering failed: ${mermaidDiagram.renderError.value} — check the template syntax in config/${activeMermaidTemplateId}.mmd and consult Mermaid docs for valid syntax.`"
      />
      <div v-else class="p-6 bg-white rounded border border-gray-200 min-h-96">
        <div :id="mermaidDiagram.containerId.value" style="display: flex; justify-content: center;"></div>
      </div>
    </template>
    ```

    Also update the MetrovizMap v-if condition so it doesn't show when activeVizType is 'mermaid':
    Current: `v-if="(activeVizType === 'metro' || !isFlowEligible) && isMetroEligible"`
    Becomes: `v-if="activeVizType !== 'mermaid' && (activeVizType === 'metro' || !isFlowEligible) && isMetroEligible"`

    And update the FlowDiagram v-else-if:
    Current: `v-else-if="isFlowEligible"`
    Becomes: `v-else-if="activeVizType !== 'mermaid' && isFlowEligible"`

    **4. Script section — watch to trigger initial render**

    Add a watch that calls renderDiagram when the diagram string is ready and the viz type is mermaid:
    ```typescript
    watch(
      () => mermaidDiagram.diagramString.value,
      async (newStr) => {
        if (!newStr || activeVizType.value !== 'mermaid') return
        await nextTick()
        await mermaidDiagram.renderDiagram(mermaidDiagram.containerId.value)
      }
    )
    ```

    (The composable itself also has a watch, but having the page-level watch ensures the container div is mounted before render is called.)

    **Import additions needed at top of script:**
    ```typescript
    import { useMermaidTemplate } from '@/composables/useMermaidTemplate'
    ```
