<task type="auto">
  <name>Task 2: Configure TailwindCSS v4 in Nuxt</name>
  <files>nuxt.config.ts, assets/css/main.css</files>
  <read_first>
    - nuxt.config.ts — see current config before modifying
  </read_first>
  <action>
Update nuxt.config.ts to use the TailwindCSS v4 Vite plugin. The final nuxt.config.ts must contain:

```typescript
import tailwindcss from '@tailwindcss/vite'

export default defineNuxtConfig({
  compatibilityDate: '2025-01-01',
  ssr: true,
  runtimeConfig: {
    // Server-only: NEVER exposed to client
    notionApiToken: process.env.NOTION_API_TOKEN || '',
  },
  css: ['@/assets/css/main.css'],
  vite: {
    plugins: [tailwindcss()],
  },
  nitro: {
    preset: 'node-server',
  },
})
```

Then create assets/css/main.css with:

```css
@import "tailwindcss";

/* Metroviz SVG canvas sizing */
.metroviz-canvas svg {
  width: 100%;
  height: auto;
}
```

Create the assets/css/ directory if it does not exist.

NOTE: TailwindCSS v4 does NOT need a tailwind.config.ts or postcss.config.ts. The @import "tailwindcss" directive in main.css is the complete configuration. Do NOT create tailwind.config.ts.
  </action>
  <verify>
    <automated>grep -q "tailwindcss()" nuxt.config.ts && grep -q "assets/css/main.css" nuxt.config.ts && grep -q '@import "tailwindcss"' assets/css/main.css && echo "OK"</automated>
  </verify>
  <acceptance_criteria>
    - nuxt.config.ts contains `import tailwindcss from '@tailwindcss/vite'`
    - nuxt.config.ts contains `css: ['@/assets/css/main.css']`
    - nuxt.config.ts contains `vite: { plugins: [tailwindcss()] }`
    - assets/css/main.css exists and contains `@import "tailwindcss"`
    - assets/css/main.css contains `.metroviz-canvas svg`
    - No tailwind.config.ts file exists (v4 does not use one)
  </acceptance_criteria>
  <done>TailwindCSS v4 is configured in nuxt.config.ts with Vite plugin and main.css entry point</done>

Download the Metroviz source files from https://github.com/rstockm/Metroviz using curl or wget. Create the directory vendor/metroviz/js/ and download only the required JS files (per D-01 and D-02 — trim to v1 must-haves):

**Required files (download these):**
- metro-renderer.js — MetroRenderer class (SVG rendering engine)
- layout-engine.js — LayoutEngine class (computes positions)
- data-model.js — DataModel class (validates and normalizes input JSON)
- utils.js — parseDate() and utility functions
- color-utils.js — color palette helpers

**Do NOT download:**
- app.js — Alpine.js entry point (not needed; we wrap the renderer directly)
- dialog.js — UI dialog code (not needed)
- editor-actions.js — JSON editor UI (not needed)
- file-manager.js — file management UI (not needed)
- markdown-export.js — export functionality (not needed)
- url-state.js — URL state management (not needed)

Download command pattern:
```bash
mkdir -p vendor/metroviz/js
BASE_URL="https://raw.githubusercontent.com/rstockm/Metroviz/master/js"
for f in metro-renderer.js layout-engine.js data-model.js utils.js color-utils.js; do
  curl -sSf "$BASE_URL/$f" -o "vendor/metroviz/js/$f"
done
```

After downloading, verify each file exists and is non-empty (greater than 0 bytes).

Also download the Metroviz CSS (required by MetroRenderer for SVG styling):
```bash
mkdir -p vendor/metroviz/css
curl -sSf "https://raw.githubusercontent.com/rstockm/Metroviz/master/css/metroviz.css" -o "vendor/metroviz/css/metroviz.css" 2>/dev/null || echo "No CSS file found — continue without it"
```
