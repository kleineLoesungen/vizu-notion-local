# Phase 2 — UI Design Contract

> Visual and interaction contract for Metroviz and Vue Flow visualization pages. Generated by gsd-ui-researcher, consumed by gsd-planner and gsd-executor.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | TailwindCSS 4.x (no shadcn) |
| Preset | not applicable |
| Component library | none (project does not require component library for Phase 2) |
| Icon library | TBD in Phase 3 (Phase 2 has no icon requirements) |
| Font | System default stack (TailwindCSS default) |

**Note:** TailwindCSS is installed as part of this phase. `tailwind.config.ts` and `postcss.config.ts` must be created. No third-party registries are used.

## Spacing Scale

Declared values (multiples of 4, TailwindCSS standard):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Metroviz SVG viewport padding |
| sm | 8px | Button/input padding (Phase 3) |
| md | 16px | Page padding, section spacing |
| lg | 24px | Major layout breaks |
| xl | 32px | Top-level page margins |
| 2xl | 48px | Header/footer spacing |

**Exceptions:** None for Phase 2. Metroviz SVG layout is independent of TailwindCSS spacing — library manages internal positioning via its own viewport configuration.

---

## Typography

Phase 2 has minimal text rendering (only labels and titles on visualizations). Full typography contract deferred to Phase 3 when interactive UI is built.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Chart labels, metadata text |
| Heading | 20px | 600 | 1.2 | Page title, section headers |

**Rationale:** Phase 2 focuses on visualization rendering, not text-heavy UI. These two sizes cover page structure and chart labels. Phase 3 will expand with form labels, buttons, and detail panels.

---

## Color

**Dominant (60%):** `#ffffff` (white background)
- Usage: Page background, canvas area for visualizations
- Rationale: Visualization content should have maximum contrast against clean background

**Secondary (30%):** `#f3f4f6` (light gray)
- Usage: Card backgrounds, chart grid lines, borders, info panels
- Rationale: Subtle contrast to separate visualization containers from main background

**Accent (10%):** `#3b82f6` (blue)
- Reserved for:
  - Active state on viz type selector buttons
  - Hover state on selectable nodes (Phase 3)
  - Primary call-to-action buttons (Phase 3)
- Rationale: Standard visualization accent for interactive elements

**Destructive:** `#ef4444` (red)
- Usage: Error states, invalid data warnings (Phase 3)
- Rationale: Clear distinction for error conditions

**Note:** Metroviz library has its own SVG color palette (station colors, line colors) that are independent of this contract. Those are configured via Metroviz JSON input, not TailwindCSS tokens. See Metro Map Data Model section below for color mapping strategy.

## Component Inventory

### Page Components

**`pages/visualizations/[sourceId].vue`**
- Layout: Full-width visualization canvas with optional sidebar for controls
- Responsibilities:
  - Fetch source data from `/api/sources/:id`
  - Determine eligible viz types based on `meta.availableRoles`
  - Render MetrovizMap or FlowDiagram based on selected type
  - Handle loading states (spinner or skeleton)
  - Handle error states (error message + guidance)
- Props: `sourceId` (from route)
- Emits: None (Phase 2 is read-only)

### Visualization Components

**`components/MetrovizMap.vue`** (Metroviz wrapper)
- Wraps the vendored Metroviz library (vanilla JS, SVG output)
- Responsibilities:
  - Transform `EnrichedPage[]` data into Metroviz JSON format
  - Initialize Metroviz on `onMounted` hook
  - Re-initialize on data changes (no reactive updates)
  - Expose Metroviz event handlers (click, hover) for Phase 3 interactions
- Props:
  - `data: EnrichedPage[]` — pages to render
  - `columnMappings: ColumnMappings` — role mappings from source config
- Emits:
  - `node-selected(page: EnrichedPage)` — (for Phase 3 detail panel)

**`components/FlowDiagram.vue`** (Vue Flow wrapper)
- Wraps `@vue-flow/core` for process flow visualization
- Responsibilities:
  - Transform `EnrichedPage[]` data into Vue Flow nodes and edges
  - Initialize Vue Flow component
  - Handle node/edge layout automatically
- Props:
  - `data: EnrichedPage[]` — pages to render
  - `columnMappings: ColumnMappings` — role mappings from source config
- Emits:
  - `node-selected(page: EnrichedPage)` — (for Phase 3 detail panel)

### Utility Components

**`components/VizTypeSelector.vue`** (Phase 3, deferred)
- Radio group or tab set for switching between metro/flow
- Disabled if only one type is eligible
- Deferred to Phase 3

**`components/LoadingSpinner.vue`**
- Simple spinner (SVG or animated div)
- Used while fetching source data
- Style: Centered on canvas, gray color (`#6b7280`)

**`components/ErrorAlert.vue`**
- Error message card
- Style: Red border (`#ef4444`), light gray background (`#fee2e2`), bold red heading
- Includes retry button (Phase 3)

## Visualization Data Models

### Metro Map (Metroviz)

**Input:** `EnrichedPage[]` with roles: `date`, `next`, `parent` (optional), `tag` (optional), `title`

**Output:** Metroviz JSON structure:
```typescript
{
  zones: [
    {
      name: string (from `tag` role value)
      lines: [
        {
          name: string (from `parent` role value or default "Default Line")
          color: string (assigned automatically, see Color Assignment below)
          stations: [
            {
              name: string (from `title` role value)
              date: YYYY-MM-DD (from `date` role value)
              type?: string (optional, from secondary role)
            }
          ]
        }
      ]
    }
  ]
  global_events?: [
    {
      name: string
      date: YYYY-MM-DD
    }
  ]
}
```

**Color Assignment Algorithm:**
- Lines are assigned colors from a fixed palette (TBD during implementation; recommend 12-color palette for distinction)
- Colors repeat if more than 12 lines exist in a zone
- Colors are deterministic based on `parent` value hash to ensure consistency across re-renders

**Station Ordering:**
- Stations within a line are sorted by `date` (ascending)
- Date format validation: ISO 8601 (YYYY-MM-DD); non-conformant dates render as "Invalid Date"

**Zone Grouping:**
- If `tag` role is mapped: pages grouped by `tag` value → each group becomes a zone
- If `tag` role is not mapped: all pages go into a single default zone ("Timeline")

**Line Grouping:**
- If `parent` role is mapped: pages grouped by `parent` value → each group becomes a line
- If `parent` role is not mapped: each page becomes its own line (degenerate mode)

### Process Flow (Vue Flow)

**Input:** `EnrichedPage[]` with role: `next` (minimum), `title`

**Output:** Vue Flow nodes and edges:
```typescript
nodes: [
  {
    id: string (page ID)
    label: string (from `title` role value)
    data: { page: EnrichedPage }
    position: { x: number, y: number } (auto-layout)
  }
]
edges: [
  {
    source: string (source page ID)
    target: string (target page ID from `next` role)
  }
]
```

**Node Styling:**
- Background: `#f3f4f6` (secondary color)
- Border: `#d1d5db` (subtle gray)
- Border radius: `8px`
- Padding: `12px 16px`
- Font: 14px, weight 400

**Edge Styling:**
- Stroke: `#9ca3af` (medium gray)
- Arrow: `>` (standard Vue Flow default)

**Layout:**
- Use Vue Flow's auto-layout (dagre or similar)
- Direction: left-to-right (LTR)

## State Management (Composables, not Pinia)

**`composables/useSourceData.ts`**

- Expose: `{ selectedType, eligibleTypes, setType }`

## Metroviz Vendor Integration

**File location:** `vendor/metroviz/` (to be created during implementation)

**Vendoring approach:**
1. Clone from https://github.com/rstockm/Metroviz
2. Copy only essential source files to `vendor/metroviz/`
3. Remove unused features (themes, zoom controls)
4. Expose minimal API via `components/MetrovizMap.vue` wrapper

**API used by Phase 2:**
- Metroviz constructor: `new Metroviz(containerElement, options)`
- Data input: `metroviz.render(zonesData)` or similar (exact method TBD during vendor inspection)
- Events: Station click, hover events (passed through to Phase 3)

**Known constraints:**
- Vanilla JavaScript, no Vue reactivity — wrapper must re-initialize on data change
- SVG output — safe to render in Vue template via `v-html` or ref
- No built-in theming — colors configured via JSON input

## Loading & Error States

### Loading State
- Show centered spinner while `isLoading === true`
- Spinner color: `#6b7280` (gray)
- Spinner size: 48px

### Error State
- Show error alert card with:
  - Red icon or left border
  - Bold heading: "Failed to load visualization"
  - Body: error message + "Check your Notion token and database configuration"
  - Retry button (Phase 3)

### Empty State
- If source has no pages: Show info card with heading and next-step guidance
- If source is not viz-eligible: Show error card with mapping instructions
