NOTION_API_TOKEN=your-actual-notion-integration-token

```json
{
  "sources": [
    {
      "databaseId": "your-actual-notion-database-id",
      "name": "Test Database",
      "columnMappings": {
        "title": "Name"
      }
    }
  ]
}
```
- Replace `databaseId` with a real Notion database ID (found in the database URL after the workspace slug)
- Replace `"Name"` with the actual property name in your database (usually "Name" or "Title")
- The integration must have access to this database (share the database page with the integration in Notion)

<task type="checkpoint:human-verify">
  <name>Verify: End-to-end pipeline integration</name>
  <files></files>
  <action>
Run each step below and confirm the expected output. This verifies the complete Phase 1 backend pipeline against a live Notion integration.
  </action>
  <verify>
    <automated>MISSING — Manual verification against running Docker container required</automated>
  </verify>
  <done>All 6 steps pass: container starts, /api/sources returns data, /api/sources/:id returns enriched pages, caching prevents duplicate calls, token is not visible in browser, and invalid config causes a useful error.</done>
  <how-to-verify>
Run these steps in order:

**Step 1 — Build and start the container:**
```bash
docker-compose up --build
```
Expected: Build completes (both stages). Container starts. Logs show:
- `[vizu] Loaded config with N source(s) from /app/config/sources.json`
- Nitro server listening on port 3000

**Step 2 — Verify sources list endpoint:**
```bash
curl http://localhost:3000/api/sources
```
Expected: JSON response like:
```json
{
  "sources": [
    {
      "id": "your-database-id",
      "name": "Test Database",
      "databaseId": "your-database-id",
      "columnMappings": { "title": "Name" }
    }
  ]
}
```

**Step 3 — Verify source data endpoint:**
```bash
curl http://localhost:3000/api/sources/your-database-id
```
Expected: JSON response with `source`, `pages` (array of Notion pages), and `meta.total > 0`.
Each page in `pages` has a `resolvedRelations` field (empty object `{}` if no relations).

**Step 4 — Verify caching works:**
```bash
curl http://localhost:3000/api/sources/your-database-id
```
Run immediately after Step 3. Container logs should show NO new Notion API calls. Response should be identical to Step 3.

**Step 5 — Verify token is not exposed:**
Open browser DevTools → Network tab → visit http://localhost:3000/api/sources.
Inspect the response and all request/response headers. `NOTION_API_TOKEN` or the actual token value must NOT appear anywhere.

**Step 6 — Verify config validation catches errors:**
```bash
# Stop the container
docker-compose down

# Temporarily break sources.json
echo '{"invalid": true}' > config/sources.json

# Restart — should show error on first request
docker-compose up

# Restore good config
cp config/sources.example.json config/sources.json
# (then restore your actual databaseId and columnMappings values)
```
Expected: Container starts but the first request returns a 503 with `[vizu] Config schema validation failed` message — NOT a generic 500 or silent failure.
  </how-to-verify>
  <resume-signal>
Type "approved" if all 6 steps succeed.
Type "issues: [description]" if any step fails — describe what went wrong and what error appeared.
