# Phase 5 — UI Design Contract

> Visual and interaction contract for Mermaid diagram template rendering. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | HTML SVG inline (no external icon library) |
| Font | System default (via TailwindCSS) |

**Rationale:** Project uses TailwindCSS v4 directly (no shadcn). All existing UI components (MetrovizMap, FlowDiagram, FilterPanel, ErrorAlert, LoadingSpinner) use inline TailwindCSS utility classes. Mermaid renders pure SVG client-side, no custom TailwindCSS-styled wrapper needed beyond container.

## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px (p-4, px-4) | Default element spacing |
| lg | 24px (p-6, gap-6) | Section padding, standard gap |
| xl | 32px | Layout gaps (gap-8) |
| 2xl | 48px | Major section breaks (gap-12) |
| 3xl | 64px | Page-level spacing (gap-16) |

**Mermaid diagram container:** `p-6` (24px padding) to match existing viz container padding in `[sourceId].vue`. No custom spacing exceptions.

Exceptions: none

| Role | Size | Weight | Line Height |

## Color

Inherited from existing project palette:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff (white) | Page background, viz container |
| Secondary (30%) | #f3f4f6 (gray-100), #e5e7eb (gray-200) | Card backgrounds, borders, hover states |
| Accent (10%) | #2563eb (blue-600) | Interactive elements: links, active state, CTA buttons |
| Destructive | #ef4444 (red-600) | Error alerts, validation failures |

**Accent reserved for:**
- CTA buttons (e.g., "Refresh" in SourceCard)
- Active state indicators (e.g., selected source in dropdown)
- Hover states on interactive elements
- Success feedback (e.g., "Copied!" link button turns green-600)

**Mermaid-specific:** Mermaid library renders diagrams with its own color scheme (internal SVG styling). No TailwindCSS color overrides applied to Mermaid output. Error container uses existing red-600/red-100 pattern from ErrorAlert component.

## Copywriting Contract

### Viz Type Selector Label
- **Element:** Viz type selector button in page header (alongside existing "Metro" / "Flow" options)
- **Copy pattern:** Template title from frontmatter (e.g., "Project Timeline", "Gantt Schedule")
- **Rationale:** D-08 from CONTEXT.md — template title in frontmatter becomes the viz type label in selector

### Mermaid Diagram Rendering States

| State | Heading | Body | Action |
|-------|---------|------|--------|
| Loading | (uses LoadingSpinner component) | — | Existing component, no new copy |
| Render Error | Failed to render diagram | `Template rendering failed: {error-message}` — check the template syntax in `config/{filename}.mmd` and consult [Mermaid docs](https://mermaid.js.org/intro/syntax-reference.html) for valid syntax. | User can edit template file and restart container |
| Fetch Error | Failed to load diagram data | `Could not fetch data for diagram: {source-name} — {error-message}`. Check that sources are configured correctly in `sources.json` and Notion integration is accessible. | Likely Notion API issue; verify token and source configuration |
| Empty State | No items to diagram | The source `{source-name}` contains no pages. Diagrams require at least one page with mapped properties. | User can add pages to Notion database or adjust filters |
| Validation Error (startup) | — | Console error only: `[vizu] Template loading failed: {template-filename} — {reason}. Available sources: {list}` | Admin reviews template frontmatter and sources.json, restarts container |

**Example render error:** "Failed to render diagram: Template rendering failed: unexpected token at line 5 — check the template syntax in config/project-timeline.mmd and consult Mermaid docs for valid syntax."

### Destructive Actions
None in Phase 5. (Template editing/deletion happens via file system, not UI.)

## Integration Points (UI Changes to Existing Pages)

### 1. Viz Type Selector Extension
**Page:** `pages/visualizations/[sourceId].vue` (existing)

**Current state:**
- Viz type selector renders "Metro" and "Flow" options dynamically based on eligibility
- Existing code pattern in template:
  ```html
  <div class="flex items-center gap-2">
    <span class="text-xs text-gray-500">Diagram:</span>
    <button v-if="isMetroEligible" ...>Metro</button>
    <button v-if="isFlowEligible" ...>Flow</button>
  </div>
  ```

**Change:** Add Mermaid template options in same selector
- For each template in `mermaidTemplates` (from `useSourceData` composable), render a button with template title
- Template ID passed to Mermaid render endpoint on click
- Active viz type styling: `text-blue-600 bg-blue-50` for selected, `text-gray-500 hover:text-gray-900` for unselected (existing pattern)

**Copy:** Use template title from frontmatter as button label. No verb prefix ("Metro", "Flow", "Project Timeline" — all are noun labels).

### 2. Mermaid Diagram Container
**Location:** In viz switch statement, case `activeVizType === 'mermaid'`

**Styling:** Match existing viz container dimensions and padding
```html
<div class="p-6 bg-white rounded border border-gray-200 min-h-96">
  <!-- Mermaid diagram renders here -->
  <div id="mermaid-{templateId}" style="display: flex; justify-content: center;"></div>
</div>
```

**Baseline spacing:**
- Container padding: `p-6` (24px) to match MetrovizMap container in existing code
- Min height: `min-h-96` (384px) for reasonable diagram viewport
- Border: `border border-gray-200` (thin gray line, matches card pattern)

### 3. Error Alert for Mermaid Failures
**Component:** Reuse existing `ErrorAlert` component (heading + message)

**Styling:** No changes needed — component already styled with red-600/red-100 pattern:
```html
<ErrorAlert
  heading="Failed to render diagram"
  message="Template rendering failed: {error}. Check template syntax in config/{filename}.mmd."
/>
```

## Interaction Contract

### Mermaid Diagram Click Behavior
**Default:** Mermaid library controls all internal click/zoom/pan interactions (default browser SVG behavior). No custom Node Detail Panel click handlers needed for Mermaid diagrams in v1.

**Rationale:** Unlike Metro/Flow which have internal node click detection, Mermaid diagrams are pure rendered SVG from the library. Extending click-to-detail for Mermaid diagrams requires parsing Mermaid internal node IDs (library-specific), deferred to v2.
