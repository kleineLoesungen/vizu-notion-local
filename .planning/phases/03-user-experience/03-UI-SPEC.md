# Phase 3 — UI Design Contract: User Experience

> Visual and interaction contract for Phase 3 frontend. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase Scope:** Interactive UI for exploring, filtering, and sharing visual diagrams. Covers: dashboard entry page with source management, multi-source metro map composition, per-node filtering, SVG/PNG export, Notion deep links, and URL-shareable state.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | TailwindCSS v4 (no config file) |
| Preset | not applicable |
| Component library | none (custom Vue components only) |
| Icon library | none (inline SVG only) |
| Font | System stack (TailwindCSS default) |

**Rationale:** No shadcn detected in codebase; RESEARCH.md explicitly requires no new npm dependencies for Phase 3 v1. TailwindCSS v4 with `@import "tailwindcss"` in CSS provides full utility coverage. Icon library deferred to v2.

## Spacing Scale

Declared values (multiples of 4 as per TailwindCSS standard):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight borders |
| sm | 8px | Compact element spacing, label gaps |
| md | 16px | Default button/input padding, card padding |
| lg | 24px | Section padding, filter panel internal gaps |
| xl | 32px | Layout gaps, major card spacing |
| 2xl | 48px | Major section breaks, page-level gaps |
| 3xl | 64px | Top-level page padding |

**Exceptions:** None.

**Touch targets:** Minimum 44px height for clickable elements (buttons, checkbox labels, filter controls). Implement via `py-2` + `px-3` for compact buttons, `py-3` + `px-4` for primary CTAs.

## Interaction Contract

### Source Navigation (UI-01)

- **Source Selection:** Click on a source name → Navigate to `/visualizations/{sourceId}`
- **Viz Type Selection (Phase 2 pattern):** When both metro and flow are eligible, click type button → Switch visualization in place (no page reload)
- **Back Navigation:** Click "← Back to sources" → Navigate to `/` (dashboard)

### Source Toggle (UI-02)

**Scope Note:** Phase 3 CONTEXT.md does not require "enable/disable sources from view" (multi-source visualization) for v1. D-06/D-07 are deferred. Current interaction is single-source navigation only. **No UI-02 implementation in v1.**

### Visualization Type Selector (UI-03)

- **Current:** Implemented in Phase 2 `[sourceId].vue`. Buttons toggle between Metro and Flow when both eligible.
- **No changes needed for Phase 3.** (Existing pattern reused.)

### Filter Panel (UI-04)

- **Property Filter:** User selects a property from dropdown → Enters value → Clicks "Apply Filter"
  - Filter chip appears in "Active Filters" section
  - Visualization updates instantly (client-side computed filter on pages array)
  - URL query param updated with encoded filter state
- **Remove Filter:** Click ✕ on filter chip → Filter removed
  - Visualization updates instantly
  - URL query param updated
- **Node Visibility Toggle:** Click checkbox next to node name → Node hidden/shown in visualization
  - Checkbox state updates instantly
  - Visualization updates instantly (re-render with filtered pages array)
  - URL query param updated with hiddenNodes array

### Node Detail Panel (UI-05)

- **Open Detail Panel:** Click on a node in the visualization → Detail panel slides in from right
  - Panel contains all Notion properties for that page
  - "Open in Notion →" link opens Notion page in new tab
- **Close Detail Panel:** Click ✕ button on panel header → Panel slides out
  - Visualization remains unchanged; panel state is ephemeral (not URL-encoded)

### URL State Sharing (UI-06)

- **Share Button:** (New) Add "Copy Link" button on viz page
  - Encodes current state: active source ID, viz type, active filters, hidden nodes
  - Copies URL to clipboard via JavaScript `navigator.clipboard.writeText()`
  - Shows toast notification: "Link copied to clipboard"
- **Shared Link Restoration:** User receives URL → Pastes into browser
  - URL query params decoded on page load
  - Source ID parsed → Data fetched for correct source
  - Viz type parsed → Correct visualization type selected
  - Filters decoded → Filter panel populated with active filters
  - Hidden nodes decoded → Visualization renders with nodes hidden
  - Entire view is restored exactly

**URL Encoding Strategy:**
```
/visualizations/{sourceId}?vizType=metro&filters=[...JSON...]&hiddenNodes=[...JSON...]
```
- `vizType`: string ('metro' or 'flow')
- `filters`: JSON array of filter criteria (URL-encoded)
- `hiddenNodes`: JSON array of page IDs (URL-encoded)
- **Example:** `/visualizations/abc123?vizType=metro&filters=%5B%7B%22propertyName%22%3A%22Status%22%2C%22value%22%3A%22Draft%22%7D%5D&hiddenNodes=%5B%22page-id-1%22%5D`

## Export Behavior (D-09)

**SVG Export:**
- Serializes the SVG DOM element (Metroviz or Vue Flow canvas) via `XMLSerializer`
- Creates a Blob with type `image/svg+xml;charset=utf-8`
- Triggers browser download with filename: `visualization-{vizType}-{timestamp}.svg`
- Respects current node visibility state (only visible nodes exported)

**PNG Export (v2 — optional):**
- If `html2canvas` is installed, provide PNG button
- Converts visualization container to canvas via html2canvas
- Triggers browser download with filename: `visualization-{vizType}-{timestamp}.png`
- Deferred for v1 per RESEARCH.md (no new dependencies)
