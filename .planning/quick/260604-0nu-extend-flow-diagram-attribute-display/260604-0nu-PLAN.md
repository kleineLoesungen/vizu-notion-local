<objective>
Extend the flow diagram so users can choose which page attribute (date, responsible, status, etc.) is displayed inside each node below the title — selected from a dropdown above the diagram populated with the roles already defined in that source's columnMappings.

Purpose: Flow nodes currently show only the title, wasting the node space and hiding useful context that is already being fetched from Notion. This makes the flow diagram actionable without any config changes.
Output: Two-line flow nodes with a role picker; useFlowData gains subLabel extraction; FlowDiagram renders the second line.

Extend `FlowNode` to carry a `subLabel` field and accept an optional `nodeAttribute` parameter.

1. Update `FlowNode.data` type:
```typescript
export interface FlowNode {
  id: string
  data: { label: string; subLabel: string; page: EnrichedPage }
  position: { x: number; y: number }
}
```

2. Add `nodeAttribute?: string` as a third parameter to `useFlowData`:
```typescript
export function useFlowData(
  pages: EnrichedPage[],
  columnMappings: ColumnMappings,
  nodeAttribute?: string
): { nodes: FlowNode[]; edges: FlowEdge[] }
```

3. Add a `extractSubLabel(page, propName)` helper just before the `nodes` mapping. It must handle the same Notion property types that the codebase already handles elsewhere — cover these cases:
   - `date` type: return `(prop as any).date?.start ?? ''` (ISO date string, e.g. `"2026-03-15"`)
   - `rich_text` type: return `(prop as any).rich_text?.[0]?.plain_text ?? ''`
   - `title` type: return `(prop as any).title?.[0]?.plain_text ?? ''`
   - `select` type: return `(prop as any).select?.name ?? ''`
   - `multi_select` type: return `(prop as any).multi_select?.map((s: any) => s.name).join(', ') ?? ''`
   - `people` type: return `(prop as any).people?.map((p: any) => p.name ?? p.id).join(', ') ?? ''`
   - `checkbox` type: return `(prop as any).checkbox ? 'Yes' : 'No'`
   - `number` type: return `String((prop as any).number ?? '')`
   - `formula` type: check `f.type` — handle `string`, `date`, `number`, `boolean` subtypes; return string representation
   - default: return `''`

4. In the `nodes` mapping, compute `subLabel`:
```typescript
let subLabel = ''
if (nodeAttribute) {
  const subPropName = columnMappings[nodeAttribute]
  if (subPropName) {
    const subProp = page.properties[subPropName]
    if (subProp) subLabel = extractSubLabel(subProp)
  }
}
return {
  id: page.id,
  data: { label, subLabel, page },
  position: positions.get(page.id) ?? { x: 0, y: 0 },
}
```
