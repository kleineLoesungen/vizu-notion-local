# Quick Task 260604-1j9: Fix Metro Diagram Scroll — Remove D3 Default Wheel Handler

**One-liner:** Replaced D3's unconditional-preventDefault wheel listener with a custom handler that only intercepts Ctrl/Meta+wheel for zoom, letting normal scroll pass through to the page.

## What Was Done

**Task 1: Replace D3 wheel listener with custom handler in setupZoom()**

The root cause: D3's zoom behavior registers its wheel listener with `{ passive: false }` and calls `event.preventDefault()` on every wheel event. The `.filter()` call only controls whether the zoom _transform_ is applied — it does not prevent `preventDefault()` from being called. As a result, page scroll was blocked whenever the cursor was over the metro SVG.

Fix applied in `vendor/metroviz/js/metro-renderer.js` `setupZoom()`:

1. Removed the `.filter()` chain from `d3.zoom()` — no longer needed.
2. After `svg.call(zoom)`, added `svg.on('wheel.zoom', null)` to detach D3's built-in wheel listener.
3. Attached a custom `addEventListener('wheel', ...)` with `{ passive: false }` that:
   - Returns early (no `preventDefault`, event propagates) when neither `ctrlKey` nor `metaKey` is held.
   - Calls `preventDefault()` and manually computes and applies the zoom transform via `zoom.transform(svg, nextTransform)` when Ctrl or Meta is held.
