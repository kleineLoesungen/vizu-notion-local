more nodes are hidden than visible.

and invert-selection keep URLs short regardless of how many nodes are toggled.

Output: Single `?v=` param (compressed JSON) replaces multiple flat query params; legacy
uncompressed URLs continue to decode correctly.

  <action>
    1. Add `lz-string` to dependencies in package.json (latest stable, ~1.5.x). Also add
       `@types/lz-string` to devDependencies.

    2. Rewrite `utils/state-encoding.ts`:

    New ViewState interface — extend existing fields, add the new ones:
    ```ts
    export interface ViewState {
      vizType: 'metro' | 'flow'
      filters: FilterCriteria[]
      hiddenNodes: string[]         // stored list (may be inverted — see invertedSelection)
      invertedSelection: boolean    // true when hiddenNodes actually stores VISIBLE ids
      selectedSourceIds: string[]   // extra metro source IDs currently selected
      sourceDisplayModes: Record<string, string>  // per-source 'milestones'|'line'
    }
    ```

    New `encodeViewState` signature: `encodeViewState(state: ViewState, totalNodes: number): Record<string, string>`
    - Compute invert: if `state.hiddenNodes.length > totalNodes / 2`, use invert — store
      visible IDs (pages NOT hidden) instead, set `invertedSelection: true`.
    - Build the full state object with the (potentially inverted) node list.
    - Serialize to JSON, compress with `LZString.compressToEncodedURIComponent(json)`.
    - Return `{ v: compressedString }` — single param only.
    - Omit defaults to keep payload small: skip `vizType` key if 'metro', skip empty arrays,
      skip `invertedSelection: false`, skip empty objects for selectedSourceIds/sourceDisplayModes.

    New `decodeViewState`:
    - Check `query.v` first. If present: decompress with
      `LZString.decompressFromEncodedURIComponent(query.v as string)`, parse JSON.
      Reconstruct full ViewState, defaulting missing keys (vizType→'metro', arrays→[],
      invertedSelection→false, etc.).
    - Legacy fallback (no `v` param): keep exact existing logic reading `query.vizType`,
      `query.filters`, `query.hiddenNodes` so old shared links still work.
    - `invertedSelection` is NOT applied here — caller (viz page onMounted) handles inversion
      using total page count. Return decoded state as-is; the viz page will convert visible→hidden
      list when `invertedSelection === true`.

    Import: `import LZString from 'lz-string'`
  </action>

    TypeScript check: `npx nuxi typecheck` passes (or at minimum no new errors in state-encoding.ts).
  </verify>
  <done>

    decodeViewState handles compressed `v` param and falls back to legacy flat params.
