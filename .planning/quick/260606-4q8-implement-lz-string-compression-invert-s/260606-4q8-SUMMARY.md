# Quick Task 260606-4q8: LZ-String Compression + Invert-Selection for Share Link

**One-liner:** LZString compression reduces share URLs to a single `?v=` param with invert-selection minimising payload when most nodes are hidden.

---

## What Was Built

### Task 1 — Install lz-string, expand ViewState (b49f033)

- Added `lz-string ^1.5.0` to `dependencies` and `@types/lz-string ^1.5.0` to `devDependencies` in `package.json`.
- Rewrote `utils/state-encoding.ts`:
  - Extended `ViewState` interface with `invertedSelection: boolean`, `selectedSourceIds: string[]`, `sourceDisplayModes: Record<string, string>`.
  - `encodeViewState(state, totalNodes)` serializes state to JSON, compresses via `LZString.compressToEncodedURIComponent`, returns `{ v: compressedString }`. Defaults omitted (vizType 'metro', empty arrays/objects, `invertedSelection: false`).
  - `decodeViewState` checks for `query.v` first — decompresses + parses JSON, fills missing keys with defaults. Falls back to legacy flat params (`vizType`, `filters`, `hiddenNodes`) for backward compat.

### Task 2 — Wire new fields into handleCopyLink and restore logic (af5384b)

- Updated `composables/useUrlState.ts`: `pushState` and `copyShareLink` now require `totalNodes` as a second argument.
- Updated `pages/visualizations/[sourceId].vue`:
  - `handleCopyLink`: computes invert-selection — when `hiddenNodes.length > totalNodes / 2`, stores visible IDs instead and sets `invertedSelection: true`. Includes `selectedSourceIds` (extra sources only, not primary) and `sourceDisplayModes` filtered to active extra sources.
  - `onMounted` restore: all page-dependent restore logic moved into `watch(pages, ...)` callback. Applies inversion on restore (`invertedSelection === true` → compute hidden as complement of stored visible list). Restores `selectedSourceIds` and `sourceDisplayModes` after pages load.

---

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| Inversion computed in handleCopyLink, not encodeViewState | encodeViewState lacks allPageIds needed to compute the visible complement; handleCopyLink has full pages list |
| encodeViewState receives pre-inverted state + invertedSelection flag | Clean separation: encoder just serializes; caller decides representation |
| totalNodes kept as encodeViewState param | API symmetry — may be used for future optimisations; harmless no-op currently |
| Legacy flat-param fallback preserved | Old share links remain functional without migration |

### Auto-fixed Issues

**1. [Rule 1 - Bug] `encodeViewState` cannot compute visible-ID complement internally**

- **Found during:** Task 1 implementation
- **Issue:** Plan spec says encodeViewState should "store visible IDs (pages NOT hidden)" when inverting, but the function only receives `hiddenNodes` and `totalNodes` — not all page IDs. Computing the complement is impossible without the full ID list.
- **Fix:** Moved inversion logic to `handleCopyLink` in the viz page, which has access to `pages.value`. `encodeViewState` receives the already-inverted state (caller sets `invertedSelection: true` and passes visible IDs as `hiddenNodes`). Result is identical: URL stores the shorter list.
- **Files modified:** `utils/state-encoding.ts`, `pages/visualizations/[sourceId].vue`
