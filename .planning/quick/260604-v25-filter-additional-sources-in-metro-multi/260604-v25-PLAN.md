<objective>
Filter the "Additional Sources" checkbox list on the metro viz page so it only shows sources that can meaningfully overlay on the metro map.

Purpose: Sources with no `date` role cannot be positioned on a timeline axis — showing them as checkboxes is misleading because selecting them produces no visible output.

Output: A `eligibleAdditionalSources` computed ref that filters `allSources` to sources where `columnMappings` contains at least the `date` role (milestone overlay), excluding the current primary source. The template `v-for` loop uses this computed instead of raw `allSources`.
</objective>

  <action>
In the `<script setup>` block, after the `allSources` computed (line ~347), add a new computed:

```ts
// Filter additional-source candidates to those eligible as metro overlays:
// - must have 'date' role (milestone overlay, positioned on the timeline axis)
// - must not be the current primary source
// Sources without 'date' cannot be meaningfully displayed on the metro map.
const eligibleAdditionalSources = computed(() =>
  allSources.value.filter(src =>
    src.id !== sourceId.value && 'date' in (src.columnMappings ?? {})
  )
)
```

Note: `allSources` items already carry `columnMappings` because `sources.get.ts` includes it in the list response. No additional API calls needed.

Then in the template, update the multi-source selector section (around line 179–197) in two places:

